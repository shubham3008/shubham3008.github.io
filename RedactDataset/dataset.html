How was the dataset collected?
This data was collected under the START (Screening Tools for Autism Risk using Technology) Initiative. An app was developed under the same name which helped non-specialists in autism diagnosis. A total of 131 children from ages 2-7 from the low resources settings in the Delhi-NCR region were observed. Of them, 48 were diagnosed with Autism Spectrum Disorder, 43 with Intellectual Disabilities and the rest were typically developing. The primary reason for gathering these videos was a Preferential Looking Task conducted as part of the assessment. The purpose of this task was to capture gaze location of the candidates in response to the stimuli from the app as an indication of their social preference.

1. You can use just the videos (no scripts, no eye overlay) for your tasks tasks (e.g. emotion detection) by a third party. It then functions as a data repository.

2. Same data with eyes with some metadata (e.g. gender prediction)

3. Script to redact videos provided a third party has a separate face detector bounding box for his/her own videos.

4. Our tool to create bounding boxes (multiple bounding boxes) which works in "difficult environment" such as Indian "chawl" conditions, poor lighting, low res, multiple humans (such as caregiver hovering around the child) and so on.

5. A "neural network model" which when provided with a "face" or a "redacted face", can then be used to classify gaze into left and right. The model is accompanied by the code that needs to be run. Each version will require a “license” acceptance as per the funding sponsor from the UK
